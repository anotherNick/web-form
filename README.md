# TOP Form Styling and Validation

This page is designed based off of the design file provided for Project: Sign-up Form in the Intermediate HTML and CSS course. [Design File](https://cdn.statically.io/gh/TheOdinProject/curriculum/afdbabfab03fbc34783c6b6f3920aba4a4d3b935/intermediate_html_css/forms/project_sign_up_form/imgs/sign-up-form.png)

I followed their layout somewhat closely while using my own images and color palette. The intent was to create a unique page while respecting the design provided, including decisions like matching the bottom of the logo ribbon to the top of the form, for example.

## Assignment Requirements

- Base your page on the design file.
- Use at least 2 custom fonts.
- Input elements have a thin, colored border.
- Input border changes color on focus with a light box shadow.
- Input border turns red with :invalid information.
- Do not design for mobile.
- Include html5 validation, exclude JavaScript validation.
- Button is styled similarly to design file

In the interest of exploring CSS techniques, I have set the password field to have a red outline and display a span with the rules for password creation any time a user's password is not valid. In a real site where accessibility matters, I would want that information displayed clearly in a place where a screen reader would access it before encountering the input box.
